# WorkLedger Mockup

## Scope

이 문서는 `workledger-mvp`의 Flutter Android 구현 전 화면 구조와 상태별 UI 문구를 고정한다. 고해상도 시안이 아니라 실제 위젯 구현에 바로 옮길 수 있는 문서형 mockup이다.

근거 파일은 다음이다.

- `docs/02-design/features/workledger-mvp.design.md`
- `docs/02-design/design-system-rules.md`
- `.lazyweb/design-research/workledger-mobile-time-tracking-2026-06-12/report.html`

## Design Direction

Lazyweb 리서치의 결론에 따라 홈 화면은 기능 목록이 아니라 `지금 출근 중인가?`를 1초 안에 판단하게 하는 상태 중심 화면으로 둔다. Airtable 기반 디자인 규칙은 WorkLedger에 다음처럼 적용한다.

| Rule | WorkLedger 적용 |
|---|---|
| 흰 캔버스 | 모든 기본 화면의 배경은 흰색이다 |
| 진한 잉크 CTA | 화면당 하나의 주요 버튼만 near-black primary CTA로 둔다 |
| 절제된 카드 | 카드에는 1px hairline 또는 soft surface만 사용하고 과한 그림자는 쓰지 않는다 |
| 제한적 시그니처 표면 | 월간 요약, 잔여연차, 가격 intent처럼 강조가 필요한 영역에만 코럴/포레스트/다크 표면을 사용한다 |
| 가격 sub-system | 가격 fake-door 화면의 plan CTA만 pill 형태를 허용한다 |
| SaaS 팀 관리 회피 | 승인, 팀원, 프로젝트, 위치, 급여 정확성 UI를 노출하지 않는다 |

## Mobile Frame

```text
width: 360-430dp
layout: single column
surface: white canvas
top bar: 56dp, text only or simple icon button
primary action: one near-black CTA per viewport
secondary action: white outline or text action
card radius: 10-12dp
input radius: 6dp
```

## S-01 Home / Before Clock-In

목표는 앱 실행 후 바로 출근 기록을 완료하는 것이다. 출근 전 상태에서는 `출근하기` 하나만 primary CTA로 보인다.

```text
+--------------------------------+
| 내근무장부              [설정] |
+--------------------------------+
| 오늘 · 6월 12일 금요일         |
| 개인 로컬 기록                 |
|                                |
| ┌────────────────────────────┐ |
| | 아직 출근 전                | |
| | 기록된 근무 시간이 없습니다 | |
| | 회사명·위치 없이 시작합니다 | |
| └────────────────────────────┘ |
|                                |
| [출근하기]                     |
|                                |
| 이번 달                        |
| ┌────────────┐ ┌────────────┐ |
| | 총 근무     | | 남은 연차   | |
| | 84시간 20분 | | 7.5일       | |
| └────────────┘ └────────────┘ |
|                                |
| [월간 요약]        [연차 관리] |
+--------------------------------+
```

Style notes:

- `출근하기`는 near-black primary CTA다.
- 이번 달 요약 카드는 흰 카드와 hairline만 사용한다.
- 로컬/무계정 문구는 신뢰 보조 문구로 짧게 둔다.

## S-02 Home / Working

근무 중 상태에서는 현재 상태와 경과 시간을 가장 크게 보여준다. 주요 CTA는 `퇴근하기` 하나다.

```text
+--------------------------------+
| 내근무장부              [설정] |
+--------------------------------+
| 오늘 · 6월 12일 금요일         |
|                                |
| ┌────────────────────────────┐ |
| | 근무 중                     | |
| | 출근 09:03                  | |
| | 현재 3시간 42분 기록 중     | |
| └────────────────────────────┘ |
|                                |
| [퇴근하기]                     |
|                                |
| 빠른 보정                      |
| [메모 추가]        [오늘 수정] |
|                                |
| 태그                           |
| [야근] [퇴근 지연] [휴일근무]  |
+--------------------------------+
```

Style notes:

- 경과 시간은 큰 숫자보다 명확한 문장으로 표시한다.
- 태그는 primary CTA보다 작고 조용한 chip으로 둔다.
- 팀/프로젝트/고객 선택 UI는 추가하지 않는다.

## S-03 Home / After Clock-Out

퇴근 후 상태에서는 완료감을 주되, 다음 행동은 수정 또는 월간 확인으로 낮춘다.

```text
+--------------------------------+
| 내근무장부              [설정] |
+--------------------------------+
| 오늘 · 6월 12일 금요일         |
|                                |
| ┌────────────────────────────┐ |
| | 오늘 기록 완료              | |
| | 09:03 - 18:42               | |
| | 총 9시간 39분               | |
| └────────────────────────────┘ |
|                                |
| 참고 태그                      |
| [퇴근 지연 선택됨] [야근]      |
|                                |
| 메모                           |
| 배포 대응 후 퇴근              |
|                                |
| [오늘 기록 수정]               |
| [월간 요약 보기]               |
+--------------------------------+
```

Style notes:

- 완료 카드에는 성공색을 넓게 쓰지 않는다. 흰 캔버스와 진한 잉크 문구로 충분히 표현한다.
- `월간 요약 보기`는 secondary CTA다.

## S-04 Edit Today Record

수동 수정은 숨기지 않는다. 리서치 기준으로 기록 신뢰성을 보강하는 핵심 보조 흐름이다.

```text
+--------------------------------+
| 오늘 기록 수정           [저장] |
+--------------------------------+
| 근무일                         |
| [2026-06-12]                   |
|                                |
| 출근 시각                      |
| [09:03]                        |
|                                |
| 퇴근 시각                      |
| [18:42]                        |
|                                |
| 태그                           |
| [야근] [퇴근 지연 선택됨]      |
| [휴일근무]                     |
|                                |
| 메모                           |
| [배포 대응 후 퇴근           ] |
|                                |
| [저장]                         |
+--------------------------------+
```

Validation states:

```text
+--------------------------------+
| 저장할 수 없습니다             |
| 퇴근 시각은 출근 시각보다      |
| 빠를 수 없습니다.              |
| [확인]                         |
+--------------------------------+
```

Style notes:

- 입력은 44dp 이상 높이의 white input + hairline border다.
- 저장 실패 시 입력값을 유지한다.

## S-05 Monthly Summary

월간 요약은 차트보다 숫자 카드가 먼저다. 첫 구현 범위는 이미 구현된 `WorkRecordRepository.findByMonth`, `calculateMonthlySummary`, `LeaveSummary` 계산으로 만들 수 있는 근무 기록과 연차 요약에 한정한다.

```text
+--------------------------------+
| 월간 요약              [2026-06]|
+--------------------------------+
| ┌────────────────────────────┐ |
| | 이번 달 총 근무             | |
| | 168시간 30분                | |
| | 개인 참고용 기록입니다      | |
| └────────────────────────────┘ |
|                                |
| ┌────────────┐ ┌────────────┐ |
| | 근무일      | | 초과 참고   | |
| | 21일        | | 22시간 10분 | |
| └────────────┘ └────────────┘ |
|                                |
| 태그별 참고                    |
| 야근                 12시간     |
| 퇴근 지연             4시간     |
| 휴일근무              6시간     |
| 급여 계산이 아닌 개인 참고용    |
|                                |
| 이번 달 기록                   |
| 06-03 09:02-20:15  야근        |
| 06-04 09:10-18:22              |
| 06-10 출근만 기록됨            |
|                                |
| [홈으로]                       |
+--------------------------------+
```

Style notes:

- `이번 달 총 근무` 카드는 dark surface 후보지만 한 화면에 하나만 사용한다.
- 첫 구현에서는 `이번 달 총 근무`, `근무일`, `초과 참고`, `태그별 참고`, `연차 요약`, `이번 달 기록`을 표시한다.
- incomplete 기록은 숨기지 않고 `출근만 기록됨`, `퇴근만 기록됨`처럼 상태를 짧게 표시한다.
- 임금 산정처럼 보이는 단어는 피하고 `초과 참고`로 표시한다.
- `태그별 참고`는 야근, 퇴근 지연, 휴일근무를 구분하되 급여 계산이 아닌 개인 참고용으로 안내한다.
- `월간 리포트 만들기`는 가격 intent 저장 흐름으로 연결한다.

## S-06 Leave Balance And Usage

연차 화면은 자동 법정 계산기가 아니라 사용자가 입력한 총량과 사용 내역을 보여주는 장부다.

```text
+--------------------------------+
| 연차 관리                      |
+--------------------------------+
| 기준 연도                      |
| [2026]                         |
|                                |
| ┌────────────────────────────┐ |
| | 남은 연차                   | |
| | 11일 4시간                  | |
| | 총 15일 · 사용 3일 4시간    | |
| └────────────────────────────┘ |
|                                |
| 총 연차                        |
| [15일 0시간]                   |
|                                |
| [연차 사용 추가]               |
|                                |
| 사용 내역                      |
| 06-03  1일      개인 일정      |
| 06-10  4시간    오전 반차      |
| 06-24  1일      휴식           |
+--------------------------------+
```

Style notes:

- `남은 연차` 카드에 forest surface를 제한적으로 사용할 수 있다.
- `총 연차`는 사용자가 직접 입력한 값임을 화면 문구에서 드러낸다.
- 입사일, 회계연도, 법정 자동 계산 UI는 추가하지 않는다.

## S-07 Pricing Fake-Door

가격 화면은 실제 결제가 아니라 의향 측정이다. 가격 sub-system으로만 pill CTA를 사용한다.

```text
+--------------------------------+
| 월간 리포트                    |
+--------------------------------+
| 내 근무 기록을 월말에 정리하는 |
| 리포트 기능을 준비 중입니다.   |
| 실제 결제는 진행되지 않습니다. |
|                                |
| ┌────────────────────────────┐ |
| | Report Pass                 | |
| | 월간 리포트 1회             | |
| | 개인 기록 요약              | |
| | [관심 있음]                 | |
| └────────────────────────────┘ |
|                                |
| ┌────────────────────────────┐ |
| | Pro                         | |
| | 매월 리포트와 고급 요약     | |
| | 가격 보기 클릭 측정         | |
| | [관심 있음]                 | |
| └────────────────────────────┘ |
|                                |
| 법률 자문이나 증거 효력을      |
| 보장하지 않습니다.             |
+--------------------------------+
```

Style notes:

- 가격 plan 카드는 pricing sub-system으로 보고 pill CTA를 허용한다.
- 코럴 표면은 `리포트 기능 준비 중` 안내 카드처럼 intent를 강조하는 곳에만 쓴다.
- 결제 완료, 영수증, 구독 관리 UI는 만들지 않는다.

## Implementation Notes

- 화면 구현 시 `ThemeData`의 기본 surface는 white canvas로 둔다.
- primary CTA는 한 화면에 하나만 둔다.
- 월간 요약, 연차, 가격 fake-door 외 화면에는 코럴/포레스트/다크 표면을 쓰지 않는다.
- 설정 화면은 기존 `screen-list.md`에 남기되, MVP 핵심 mockup에서는 보조 화면으로 취급한다.
- UI 문구는 한국어를 기본으로 작성하고, 영어 문자열 구조는 구현 단계에서 분리한다.
